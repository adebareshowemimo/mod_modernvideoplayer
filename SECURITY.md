# Security Policy

## Supported versions

Security fixes are provided for the latest minor release line. During the
alpha/beta period (pre-1.0), only the latest tag is supported.

| Version | Supported |
|---------|-----------|
| 0.x     | :white_check_mark: (latest tag only) |

## Reporting a vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

Instead, email a full description to:

> `security@agunfoninteractivity.com`

Include:

- Affected version(s) and Moodle version
- Step-by-step reproduction
- Impact assessment (what an attacker could do)
- Any proof-of-concept code or screenshots
- Your preferred credit wording (or "anonymous")

### What to expect

| Milestone | SLA |
|-----------|-----|
| Acknowledgement of your report | within 72 hours |
| Initial triage + severity assessment | within 7 days |
| Fix in private branch + advisory draft | within 30 days (severity permitting) |
| Coordinated public disclosure | after fix is released |

We follow Moodle's general release cadence for security fixes and will
coordinate with Moodle HQ if the issue also affects core.

### Safe harbour

We will not pursue legal action against researchers who:

- Act in good faith
- Avoid privacy violations, data destruction, and service interruption
- Give us reasonable time to fix the issue before public disclosure
- Do not exploit the vulnerability beyond what is necessary to prove it

Thank you for helping keep Moodle sites safe.
